# Colours

<p class="standfirst">The Canopy colour palette is built on L&G’s brand colours and colour architecture, providing a clear, structured foundation for colour across all digital products. This palette ensures that every interface feels consistent, accessible, and unmistakably L&G, regardless of platform or product.</p>

## Principles

The Canopy colour system is governed by these principles:

1. **Use only approved tokens**
You must use only the tokens specified on the relevant component guidance pages. Do not create or use custom tokens. If you are contributing a new component to Canopy, consult the Canopy team for guidance on which tokens to use.

2. **Clarity and accessibility**
Colours must support clear communication and legibility. All colour combinations are designed to meet or exceed WCAG 2.2 AA standards for contrast.

3. **Consistency**
Colour usage must be consistent across all products and platforms. Design tokens and modes ensure the same colour roles are applied everywhere, reinforcing brand identity. You must not mix colour modes within a single view—each card, panel, content area, or page must use only one mode.

4. **Favour neutral backgrounds and embrace white space**
Off-white and pure white are the default backgrounds, supporting clarity and accessibility. White space is a key brand device and should not be shied away from. Limit the use of bold backgrounds to avoid visual fatigue and accessibility issues.

5. **Flat design language**
Gradients and shadows are not permitted in the overall design language.

6. **Consult brand guidelines for complex use cases**
For illustrations, photography, and charts/graphs, refer to the colour guidelines in the Brand Hub for advice on appropriate combinations.

## Usage

Canopy’s colour palette is managed through design tokens, which define specific colour roles and ensure consistent usage across all digital products. You must always use these tokens—never choose colours arbitrarily. Select modes in Figma using the <a href="https://help.figma.com/hc/en-us/articles/15343816063383-Modes-for-variables" target="_blank">Modes functionality</a>.

### Colour modes
Colour modes define the overall theme for a product or experience. Blue is the default and preferred colour mode for all Canopy products and experiences. We recommend that you lead with blue and most experiences should use blue as the primary mode. Green, Yellow, and Red colour modes may be used sparingly to inject additional brand personality, but must never be mixed within the same view. Always default to blue unless there is a clear, approved reason to use another colour mode. Each mode provides a complete set of colours for backgrounds, text, accents, and interactive elements. Modes are applied at the parent level - never mix modes within a single card, panel, content area, or page.

* **Blue** (default)

* **Green**

* **Red**

* **Yellow**

### Colour tones
Within each mode, Canopy provides four tonal themes to support different UI needs:

* **Neutral:** The default, calm tone for most use cases.

* **Neutral inverse:** A reversed version, typically for dark-on-light or light-on-dark scenarios.

* **Subtle:** Light variants for gentle brand expression.

* **Bold:** Strong, signature colours for emphasis or branding moments.

Tones must be applied purposefully to ensure clarity, hierarchy, and accessibility across backgrounds, containers, text, and interactive elements.

**Example usage**

* Use **Neutral** tones in productive environments, such as dashboards, product pages, or forms, where clarity is essential.

* Use **Subtle** tones to gently apply brand colours and highlight key messages or promotions.

* Use **Bold** tones for strong brand presence—such as marketing pages, onboarding flows, or branded apps.

*Note: Refer to the Canopy documentation or Figma library for the exact palette.*

### Status modes

Status modes are a dedicated set of colours used to communicate key states and feedback in the interface - such as information, success, warning, error, and generic messages. These colours are only used on specific components, including inline message, details, and banner.

**How status modes work**

* **Consistent across modes:**
Status colours remain the same regardless of the parent container’s colour mode (blue, green, red, yellow). This ensures that users always recognise status messages, no matter the parent container’s colour.

* **Always paired with icons:**
Statuses are always accompanied by an appropriate icon, so colour is never the sole indicator of status. This is essential for accessibility and ensures that users who cannot perceive colour differences can still understand the message.

* **Tone-aware:**
Status colours adapt to the selected colour tone (neutral, neutral inverse, subtle, bold) to maintain brand consistency and meet accessibility requirements. For example, a warning message will use a different shade on a bold background than on a neutral one, but it will always be recognisably “warning”.

* **Accessibility first:**
All status colours are designed to meet or exceed WCAG 2.2 AA contrast requirements in every tone.

### Status types

* **Info:** Used for general information and guidance.

* **Success:** Indicates successful actions or positive outcomes.

* **Warning:** Draws attention to cautionary or potentially risky situations.

* **Error:** Signals problems, failures, or required user action.

* **Generic:** For messages that don’t fit the above categories but still require user attention.

**Guidance:**

* Always use the provided status colours for their intended purpose - never substitute with brand or accent colours.

* Status colours must be applied using the correct tone for the context to ensure both brand alignment and accessibility.

* Statuses must always be paired with an appropriate icon.

* If you’re unsure about which status colour or tone to use, consult the Canopy team.

## Accessibility

All Canopy components and colour combinations are designed to meet or exceed WCAG 2.2 AA standards for colour contrast. This means:

* **Normal text** must have a contrast ratio of at least 4.5:1 against its background.

* **Large text** (18pt/24px and above, or 14pt/18.66px and above if bold) must have a contrast ratio of at least 3:1.

* **UI components** and graphical objects (such as icons, borders, and interactive elements) also meet the required contrast ratios for accessibility.

Using Canopy’s colour modes and tokens correctly will ensure these standards are maintained across all products and platforms.

It is your responsibility to check that colours are used appropriately in your designs, especially when combining elements or creating custom layouts. Always verify that your colour choices maintain sufficient contrast for readability and usability.
