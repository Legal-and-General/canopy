# Link menu

<p class="standfirst">A link menu displays a grouped list of navigation links, each with optional icons and descriptive text.</p>

![Link menu component example](docs/link-menu/hero.png)

## Usage

Use a link menu to present a set of related navigation options, typically in a sidebar, footer, or within a card.  It is best suited for grouped navigation rather than inline links or single actions.

Import the link menu components in your application:

```js
import { LgLinkMenuComponent } from '@legal-and-general/canopy';
```

and in your HTML:

<Source of={LinkMenuStories.StandardLinkMenu} />

## How it works

A link menu displays a vertical stack of menu links within a visually contained area, making it clear to users that the links form a menu. Links are visually separated for clarity and ease of scanning. Link menus work best when they contain between 2 and 10 links - fewer may not justify a menu, and more can overwhelm users.

Link menus must present navigation in a clear, purposeful structure. Group related links under meaningful section headings - especially in complex menus like footers - to help users scan and find what they need. Every link should be relevant to its group so avoid including unrelated or miscellaneous links. Link menus are not a replacement for contextual links within page content, and should only be used for grouped navigation, not as a catch-all for every possible destination.

Add a second `LgLinkMenuItemTextComponent` for additional description beneath the primary label. You can also make the primary label bold for extra clarity by setting the isBold input.

Combine with other navigation or container components as needed.

## Do and don’t

### Do

![Link menu component dos](docs/link-menu/do.png)

1. **Do** organise links into logical, related groups with clear, descriptive section headings - especially in complex menus like sidebars, navs or footers.
2. **Do** provide clear, concise labels for each link.
3. **Do** use secondary text for additional context if needed.
4. **Do** ensure icons are meaningful and support the link’s purpose.

### Don't

![Link menu component donts](docs/link-menu/dont.png)

1. **Don't** use link menus for single actions - use a link or button instead.
2. **Don't** include unrelated or miscellaneous links, or present a long, unstructured list—menus must group related links and use clear hierarchy to avoid overwhelming users with “walls of links.”
3. **Don’t** use link menus as a substitute for contextual links within page content.
4. **Don’t** overload menus with too many links or excessive secondary text.

## Developer reference

### LgLinkMenuComponent

* Projects the set of links for the menu.

### LgLinkMenuItemComponent

Projects `LgLinkMenuItemTextComponent` elements and an optional left icon.

Accepts one or two `LgLinkMenuItemTextComponents`:
* The first is the primary label.
* The second, if present, is the secondary descriptive text (always regular weight and uses muted colour tokens).

### Icon behaviour

#### Right icon (automatic):
- By default, a right-side icon is shown.
- If the parent `<a>` element has `target="_blank"`, it shows the `link-external` icon with accessible text "opens in a new tab".
- Otherwise, the `arrow-right` icon is shown.
- On hover, the icon translates 0.25rem to the right.

#### Custom icons:
- **Left icon**: Project an `<lg-icon>` component as content to display a custom icon on the left.
- **Right icon**: Use the `rightIcon` input to override the default right icon with a custom icon name, or set it to `null` to hide the right icon completely.

#### Inputs

| Name        | Description                                                          |    Type                    |   Default     | Required |
|-------------|----------------------------------------------------------------------|:--------------------------:|:-------------:|:--------:|
| `rightIcon` | Custom icon name for the right icon, or `null` to hide it completely | `string \| null \| undefined` | `undefined` |    No    |

### LgLinkMenuItemTextComponent

The text sub-component for the menu item.
Accepts a `isBold` attribute input that changes its style to make the text of the menu item bold.

When two `LgLinkMenuItemTextComponent`s are used, the second one automatically:
- Uses font weight 400 (regular) instead of 500
- Uses muted colour tokens (`--link-menu-item-rest-colour-muted`, etc.)

#### Inputs

| Name     | Description                            |    Type   |   Default   | Required |
|----------|----------------------------------------|:---------:|:-----------:|:--------:|
| `isBold` | A toggle for making the text bold      | `boolean` |   `false`   |    No    |
