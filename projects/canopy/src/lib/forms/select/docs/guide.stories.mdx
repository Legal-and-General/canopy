# Select

<p class="standfirst">A drop down list that enables the user to choose between several options.</p>

![example](docs/select/example.png)

## Usage

Use this component when users need to make a single selection from a list of options within a limited space.

Import the select component in your application:

```js
@NgModule({
  ...,
  imports: [ ..., LgSelectFieldComponent ],
});
```

and in your HTML:

## How it works

The select component controls custom select box styling and linking the label and field.

A default value can be pre-selected from the list, otherwise display an empty value of 'Select'.

The entries in the list should follow a predictable format, such as alphabetical, chronological, or most frequently used.

Like all form inputs, the width of select should match the maximum length of expected content.

When the user interacts with the select input, the select is opened and a list of options is revealed. The user can select an option by clicking on it with the mouse or using the keyboard, or tapping. The user can also scroll through the options using the up arrow or down arrow keys, or swiping.

While the select box is styled by Canopy, the select menu styling is handled by the browser being used or according to the native OS styling if on mobile or tablet.

## Dos and Don'ts

### Do

![do](docs/select/do.png)

1. **Do** use a select component when the preset options is a long list (more than five) and space is limited.
2. **Do** list options in a predictable format, such as alphabetical, chronological, or most frequently used.

### Don't

![dont](docs/select/dont.png)

1. **Don't** use a select component when typing may be faster, such as date of birth or postcode; use text input instead.
2. **Don't** use a select component as a way of presenting a single selection; use a checkbox instead.
3. **Don't** use a select component when there are a large number of options; use autocomplete instead.
4. **Don't** use capitalised text, it's better to keep entries in sentence case for easy reading.

## Additional development details

**We recommend using <a href="https://angular.io/guide/reactive-forms" target="_blank">Angular's reactive forms</a>.**

### LgSelectDirective inputs

| Name   | Description                                         | Type     | Default                       | Required |
|--------|-----------------------------------------------------|----------|-------------------------------|----------|
| `id`   | HTML ID attribute, auto generated if not provided   | `string` | `lg-select-${nextUniqueId++}` | No       |
| `name` | HTML Name attribute, auto generated if not provided | `string` | `lg-select-${nextUniqueId++}` | No       |

### LgSelectFieldComponent inputs

| Name    | Description                                                                                                            | Type      | Default                       | Required |
|---------|------------------------------------------------------------------------------------------------------------------------|-----------|-------------------------------|----------|
| `id`    | HTML ID attribute, auto generated if not provided. This will also propagate to the input `id` and form `for` attribute | `string`  | `lg-select-${nextUniqueId++}` | No       |
| `block` | Property to make the select field full width (for small screens only)                                                  | `boolean` | `false`                       | No       |
