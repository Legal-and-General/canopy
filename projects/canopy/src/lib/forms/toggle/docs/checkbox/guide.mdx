# Checkbox

<p class="standfirst">A checkbox lets the user select one or more options from a set, with a clear visual indicator showing which options are selected.</p>

![Checkbox hero](docs/forms/checkbox/hero.png)

## Usage

Use a checkbox when you want to let the user choose multiple options. Each checkbox is either checked or unchecked.

This checkbox component is implemented by `LgToggleComponent`. While `<lg-toggle>` is an available (generic) selector and is used in some existing products, **we recommend using** `<lg-checkbox>`.

Checkboxes can be used in the following ways:

- **Multi-select** – selecting any combination from a set of options.
- **Agreement** or confirmation – confirming a statement before continuing.
- **Checklist items** – marking tasks or steps as complete.

### Single checkbox

Import the toggle component in your application:
```js
import { LgToggleComponent } from '@landg/angular-components';
```

And in HTML:
```html
<lg-checkbox formControlName="confirm" value="yes">Do you agree?</lg-checkbox>
```

### Checkbox group (multi-select)

Use a checkbox group when the user is choosing from a related set of options and you want shared labelling and layout.

Import the toggle and checkbox group components in your application:

```js
import { LgCheckboxGroupComponent, LgToggleComponent } from '@legal-and-general/canopy';
```

and in your HTML:

```html
<lg-checkbox-group [inline]="inline" [focus]="focus" formControlName="colors">
  {{ label }}
  @if (hint) {
    <lg-hint>{{ hint }}</lg-hint>
  }
  <lg-toggle value="red" (blur)="checkboxBlur.emit($event)">Red</lg-toggle>
  <lg-toggle value="yellow" (blur)="checkboxBlur.emit($event)">Yellow</lg-toggle>
</lg-checkbox-group>
```

## How it works

A checkbox has two states – selected and unselected. Selecting a checkbox shows a tick to confirm the option is included. Selecting it again clears the tick and removes the option from the selection.

Most of the time, a checkbox must be accompanied by a clear, descriptive label. The label ensures users understand the purpose of the checkbox and improves accessibility for assistive technologies.

- **Label** (required): Clearly describes the checkbox’s purpose. The label must be visible and associated with the checkbox input.
- **Hint text** (optional): Provides additional guidance or context. Place the hint directly below the label.
- **“Optional” indicator** (optional): If the checkbox is not required, display an “optional” label next to the main label to clarify that selection is not mandatory.
- **Validation message** (optional): Communicates errors or required actions. Display the validation message below the hint or “optional” indicator, using clear language to explain what’s wrong and how to fix it.

All elements - label, hint, “optional” indicator, and validation message - must be consistently styled and positioned according to Canopy’s design guidelines.

For more information on best practice, consult the introduction to the form design guide.

### Variants

#### Stacked

![Checkbox stacked](docs/forms/checkbox/variant-stacked.png)

Use stacked checkboxes by default. Stacked layouts are easier to scan, work better with long labels, and reduce awkward wrapping.

#### Inline

![Checkbox inline](docs/forms/checkbox/variant-inline.png)

Use inline checkboxes only when all labels are short and the options are closely related. If the layout wraps onto multiple lines or the spacing becomes uneven, switch to stacked.

#### Size

![Checkbox size](docs/forms/checkbox/variant-size.png)

- **Default** – Use in the majority of cases.
- **Small** – Use only in very dense layouts where space is limited.

## Do and don't

### Do

![Checkbox do](docs/forms/checkbox/do.png)

1. **Do** always provide a clear, descriptive label for every checkbox.
2. **Do** use a checkbox to allow the user to choose from multiple options.
3. **Do** use a single checkbox to capture acknowledgement or consent (for example, “I have read and understood the terms and conditions”).
4. **Do** use a checkbox to confirm an action has been completed (for example, in a checklist).

### Don't

![Checkbox dont](docs/forms/checkbox/dont.png)

1. **Don't** rely on placeholder text or tooltips as a substitute for a label.
2. **Don't** use negative labels.
3. **Don't** use as part of a dark pattern.
4. **Don't** use a checkbox when the user must choose one option from a set (for example, Yes/No). Use radios instead.

## Developer reference

**We recommend using <a href="https://angular.io/guide/reactive-forms" target="_blank">Angular's reactive forms</a>.**

### LgToggleComponent

#### Inputs

| Name              | Description                                                                                                                     | Type                          | Default                       | Required |
|-------------------|---------------------------------------------------------------------------------------------------------------------------------|-------------------------------|-------------------------------|----------|
| `id`              | HTML ID attribute, auto generated if not provided                                                                               | `string`                      | `lg-toggle-${++nextUniqueId}` | No       |
| `name`            | HTML name attribute                                                                                                             | `string`                      | `null`                        | Yes      |
| `value`           | HTML value attribute                                                                                                            | `string`                      | `null`                        | Yes      |
| `focus`           | Set the focus on the checkbox                                                                                                   | `boolean`                     | `null`                        | No       |
| `ariaDescribedBy` | HTML ID for the corresponding element that describes the checkbox, if not provided it will use the hint field where appropriate | `string`                      | `null`                        | No       |
| `size`            | The size of the checkbox                                                                                                        | `'sm'` \| `'lg'`              | `'lg'`                        | No       |

#### Outputs

| Name   | Description                         | Type                  | Default                       | Required |
|--------|-------------------------------------|-----------------------|-------------------------------|----------|
| `blur` | Event emitted on blur of a checkbox | `EventEmitter<Event>` | `lg-toggle-${++nextUniqueId}` | No       |

### LgCheckboxGroupComponent

#### Inputs

| Name              | Description                                                                                                                       | Type            | Default                                     | Required |
|-------------------|-----------------------------------------------------------------------------------------------------------------------------------|-----------------|---------------------------------------------|----------|
| `id`              | HTML ID attribute, auto generated if not provided                                                                                 | `string`        | `lg-checkbox-group-id-${this.nextUniqueId}` | No       |
| `name`            | Set the name value for all checkboxes in the group, auto-generated if not provided                                                | `string`        | `lg-checkbox-group-${this.nextUniqueId}`    | No       |
| `value`           | HTML value attribute. Sets the default checked checkbox buttons, must match the values of the checkbox buttons                    | `Array<string>` | `null`                                      | No       |
| `focus`           | Set the focus on the fieldset                                                                                                     | `boolean`       | `null`                                      | No       |
| `disabled`        | Set the inner checkboxes to disabled                                                                                              | `boolean`       | `false`                                     | No       |
| `ariaDescribedBy` | HTML ID for the corresponding element that describes the checkboxes, if not provided it will use the hint field where appropriate | `string`        | `null`                                      | No       |
| `inline`          | If true, displays the checkboxes inline rather than stacked                                                                       | `boolean`       | `false`                                     | No       |
