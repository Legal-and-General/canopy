# Text input

<p class="standfirst">Text input allows the user to enter a single line of text into a form.</p>

![Text input hero](docs/text-input/hero.png)

## Usage

Use Text input when a user needs to enter a short, single-line value. For example:

1. **Contact details** – name, email address, phone number, postcode
2. **Product or account details** – reference/policy number, amounts (numeric)
3. **Search**

Import the text input component in your application:

```js
import { LgInputFieldComponent, LgInputDirective } from '@legal-and-general/canopy';
```

and in your HTML:

```html
<lg-input-field>
  Name
  <lg-hint>Please enter your name</lg-hint>
  <input lgInput size="12" formControlName="name" />
</lg-input-field>
```

### Optional fields

For optional fields, use a `<span>` element with the `lg-label--optional` class to indicate that the field is not required:

```html
<lg-input-field>
  Name
  <span class="lg-label--optional">(optional)</span>
  <lg-hint>Please enter your name</lg-hint>
  <input lgInput formControlName="name" />
</lg-input-field>
```

## How it works

The text input consists of a label and an interactive field. When a user selects the field, it becomes active, allowing them to enter or edit a single line of text using a keyboard or other input method. Users should be able to move between inputs using standard keyboard navigation (for example, Tab and Shift + Tab).

A text input **must always** have a label. The label gives the user context about what information is required and must remain available when the user is entering a value.

Hint text is optional. Use hint text only when the user needs extra guidance to complete the field (for example, a format requirement). Do not use placeholder text as a replacement for a label.

Text inputs should be set to a specific size when the expected length of the input is known (for example, a postcode). Use full width for inputs where the length is variable (for example, a name).

### Variants

#### Text prefix

![Text input prefix](docs/text-input/variant-prefix.png)

Use prefixes (for example, currency symbols) to give extra context to the input value. Add a prefix using the `lgPrefix` directive:

```html
<lg-input-field>
  Amount
  <input lgInput formControlName="amount" />
  <span lgPrefix>£</span>
</lg-input-field>
```

#### Text suffix

![Text input suffix](docs/text-input/variant-suffix.png)

Use suffixes (for example, percentages) to give extra context to the input value. Add a suffix using the `lgSuffix` directive:

```html
<lg-input-field>
  Percentage
  <input lgInput formControlName="percentage" />
  <span lgSuffix>%</span>
</lg-input-field>
```

#### Input buttons (Add-on)

![Text input add-on](docs/text-input/variant-add-on-button.png)

Buttons can be added within the input field when the action relates directly to the value, such as clearing the field or revealing a password. Use the add-on priority for icon-only buttons to inherit correct spacing without adding background or border styling.

```html
<lg-input-field>
  Name
  <input lgInput formControlName="name" />
  <button lg-button lgSuffix [iconButton]="true" priority="add-on">
    Close
    <lg-icon name="close" />
  </button>
</lg-input-field>
```

#### External buttons

![Text input external button](docs/text-input/variant-external-button.png)

Use the `lgInputFieldExternalButton` directive to place a button outside the input border. This should be used for actions that trigger a process related to the input, such as a search or submit action.

```html
<lg-input-field>
  Search
  <input lgInput formControlName="search" />
  <button lg-button lgInputFieldExternalButton priority="primary">
    Search
  </button>
</lg-input-field>
```

#### Combining elements

![Text input combining elements](docs/text-input/variant-combining-elements.png)

You can combine multiple suffixes with an external button when complex interactions are required.

```html
<lg-input-field>
  Discount
  <input lgInput formControlName="discount" />
  <span lgSuffix>%</span>
  <button lg-button lgSuffix [iconButton]="true" priority="add-on">
    Close
    <lg-icon name="close" />
  </button>
  <button lg-button lgExternalButton priority="primary">
    Submit
  </button>
</lg-input-field>
```

## Do and don't

### Do

![Text input do](docs/text-input/do.png)

1. **Do** place labels above the input. This ensures the best scannability and accessibility across all device sizes.
2. **Do** match the width of the input to the expected content. Use a width that gives a visual cue to the user (for example, a shorter field for a postcode).
3. **Do** use clear and concise labels. Labels must describe exactly what information is required.
4. **Do** use hint text for format requirements. Use it to provide extra guidance (for example, "Case sensitive") rather than relying on placeholder text.
5. **Do** use the correct input type. Ensure the appropriate keyboard is triggered on mobile devices (for example, `type="tel"` for phone numbers).

### Don't

![Text input dont](docs/text-input/dont.png)

1. **Don’t** use placeholder text as a label. Placeholders disappear when the user starts typing, which creates a high cognitive load and is an accessibility failure.
2. **Don’t** truncate label text. If a label is too long, wrap it to a new line or rewrite it for conciseness.
3. **Don’t** use a text input for pre-defined responses. If a user needs to choose from a list or make a binary choice (like Yes/No), use a select component, radio buttons, or checkboxes instead.
4. **Don’t** use "Clear" buttons for short inputs. Only use the "clear" button variant when the input is likely to be long or frequently edited (for example, a search field). Avoid them on standard fields like "Name" or "Postcode" to prevent accidental deletion.

## Additional development information

**We recommend using <a href="https://angular.io/guide/reactive-forms" target="_blank">Angular's reactive forms</a>.**

### LgInputDirective inputs

| Name   | Description                                         | Type     | Default                      | Required |
|--------|-----------------------------------------------------|----------|------------------------------|----------|
| `id`   | HTML ID attribute, auto generated if not provided   | `string` | `lg-input-${nextUniqueId++}` | No       |
| `name` | HTML Name attribute, auto generated if not provided | `string` | `lg-input-${nextUniqueId++}` | No       |

### LgInputFieldComponent inputs

| Name        | Description                                                                                                            | Type      | Default                      | Required |
|-------------|------------------------------------------------------------------------------------------------------------------------|-----------|------------------------------|----------|
| `id`        | HTML ID attribute, auto generated if not provided. This will also propagate to the input 'id' and form 'for' attribute | `string`  | `lg-input-${nextUniqueId++}` | No       |
| `block`     | Property to make the input full width (for small screens only)                                                         | `boolean` | `false`                      | No       |
| `showLabel` | Show or visually hide the label                                                                                        | `boolean` | `true`                       | No       |
