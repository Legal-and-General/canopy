# Radio

<p class="standfirst">A radio button lets a user choose one option from a set of mutually exclusive choices.</p>

![Radio hero](docs/forms/radio/hero-radio.png)

## Usage

Use a radio group when you want the user to make a single selection from **2 or more** options.

Import the radio components into your application:

```js
import { LgRadioGroupComponent, LgRadioButtonComponent } from '@legal-and-general/canopy';
```

and in your HTML:

<Source of={RadioStories.Radio} />

## How it works

Radio groups must have a label. The label must describe the question the user is answering. Like other inputs, they can also include hint text when users need extra context to choose confidently.

Each radio button contains a toggle control and a label, and must always appear in a radio group with a minimum of 2 buttons. Radio button labels can also include optional hint text.

- Selecting one radio button deselects all other buttons in the group.
- Only one option can be selected at a time.
- **Keep the list short** – radio groups should usually have **2 – 5** options. Avoid more than **7** unless you have strong evidence it’s the best pattern.
- **If you go beyond 7** – only do it when research shows users benefit from scanning all options at once, and when switching to another control would add more cognitive or interaction cost.

## Variants

#### Stacked

![Radio stacked](docs/forms/radio/variant-stacked.png)

Use stacked radio buttons by default. They are easier to scan, work better with long labels, and reduce awkward wrapping.

#### Inline

![Radio inline](docs/forms/radio/variant-inline.png)

Use inline radio buttons sparingly, only when space is lacking and all labels are short and closely related. **Inline groups must have a maximum of 3 options.** If the layout wraps onto multiple lines or spacing becomes uneven, switch to stacked.

#### Size

![Radio size](docs/forms/radio/variant-size.png)

- **Default** – Use in the majority of cases.
- **Small** – Use only in very dense layouts where space is limited.

## Dos and Don'ts

### Do

![Radio do](docs/forms/radio/do.png)

1. **Do** use radio buttons when the user must choose exactly one option from a set.
2. **Do** use stacked radio buttons by default. They are easier to scan, work better with long labels, and reduce awkward wrapping.
3. **Do** keep radio groups to 2 – 5 options where possible. Avoid more than 7 unless you have strong evidence it’s the best pattern.
4. **Do** write clear, descriptive labels. Use hint text when users need extra context to choose confidently.
5. **Do** use inline radio buttons sparingly when space is lacking – and only for up to 3 short, closely related options.

### Don't

![Radio dont](docs/forms/radio/dont.png)

1. **Don’t** use radio buttons when the user can select multiple options – use Checkbox instead.
2. **Don’t** use negative, misleading, or ambiguous labels.
3. **Don’t** use inline radio buttons for more than 3 options, or when labels wrap onto multiple lines – switch to stacked.

## Developer reference

### LgRadioGroupComponent inputs

| Name     | Description                                                                    | Type                   | Default                               | Required |
|----------|--------------------------------------------------------------------------------|------------------------|---------------------------------------|----------|
| `id`     | HTML ID attribute, auto generated if not provided                              | `string`               | `lg-radio-group-id-${nextUniqueId++}` | No       |
| `name`   | Set the name value for all inputs in the group, auto-generated if not provided | `string`               | `lg-radio-group-${nextUniqueId++}`    | No       |
| `value`  | Set the default checked radio button, must match the value of the radio button | `boolean`  | `string` | `null`                                | No       |
| `focus`  | Set the focus on the fieldset                                                  | `boolean`              | `null`                                | No       |
| `inline` | If true, displays the radio buttons inline rather than stacked                 | `boolean`              | `false`                               | No       |

### LgRadioButtonComponent

#### Inputs

| Name              | Description                                                                                                                   | Type        | Default                             | Required |
|-------------------|-------------------------------------------------------------------------------------------------------------------------------|-------------|-------------------------------------|----------|
| `id`              | HTML ID attribute, auto generated if not provided                                                                             | `string`    | `lg-radio-button-${++nextUniqueId}` | No       |
| `name`            | HTML name attribute                                                                                                           | `string`    | `null`                              | Yes      |
| `value`           | HTML value attribute                                                                                                          | `string`    | `null`                              | Yes      |
| `ariaDescribedBy` | HTML ID for the corresponding element that describes the radio, if not provided it will use the hint field where appropriate  | `string`    | `null`                              | No       |
| `size`            | The size of the radio button                                                                                                  | `RadioSize` | `lg`                                | No       |

#### Outputs

| Name   | Description                             | Type                  | Default | Required |
|--------|-----------------------------------------|-----------------------|---------|----------|
| `blur` | Event emitted on blur of a radio button | `EventEmitter<Event>` | n/a     | No       |
