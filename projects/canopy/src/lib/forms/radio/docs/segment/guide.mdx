# Segment

<p class="standfirst">A segment allows users to select a single option from a small set of mutually exclusive choices, typically used as a more prominent alternative to radio buttons.</p>

![Segment hero](docs/segment/hero.png)

## Usage

Use segments in forms when a user must make a single, mutually exclusive selection from a small list of options (between 2 and 5). They are particularly effective when:

- You want to provide a more prominent visual alternative to standard radio buttons.
- You want to ensure the user makes a conscious choice (starting with no default selection).

Import the radio components into your application:

```js
import {
   LgRadioGroupComponent,
   LgRadioButtonComponent,
 } from '@legal-and-general/canopy';
```

and in your HTML:

```html
<lg-segment-group formControlName="color">
  Colour
  <lg-segment-button value="red">Red</lg-segment-button>
  <lg-segment-button value="yellow">Yellow</lg-segment-button>
</lg-segment-group>
```

## How it works

The segment component consists of a label, an optional hint, and a series of interactive buttons. To ensure users make an intentional choice, the component should start with no option selected. Once a selection is made, the component allows only one active state at a time, so choosing a new option automatically replaces the previous selection.

To maintain clarity and usability, a segment group must contain at least two options but no more than five. If more than five choices are required, a different component like a radio group or select must be used instead.

### Variants

#### Inline

![Segment inline](docs/segment/variant-inline.png)

Segments usually sit side-by-side on larger screens. This layout is preferred when horizontal space is available and labels are short.

#### Stacked

![Segment inline](docs/segment/variant-stacked.png)

Segments should stack vertically on mobile to ensure each button remains easy to tap. The specific point at which the layout switches from inline to stacked must be determined by the length of the text labels and the available screen width.

## Do and don't

### Do

![Segment do](docs/segment/do.png)

1. **Do** use segments for mutually exclusive options where only one selection is permitted.
2. **Do** stack segments vertically on mobile devices to ensure touch targets are large enough for all users.
3. **Do** keep labels short and concise to ensure they fit within the buttons without wrapping.

### Don't

![Segment dont](docs/segment/dont.png)

1. **Don't** use segments for more than five options; use a radio group or select component instead.
2. **Don't** use segments if the user needs to select multiple items; use checkboxes instead.
3. **Don't** use segments for navigation; use tabs or a button group to move between different pages or views.

## Developer reference

**We recommend using <a href="https://angular.io/guide/reactive-forms" target="_blank">Angular's reactive forms</a>.**

### Displaying the buttons at different breakpoints

Segment buttons are displayed inline, unless given a `RadioStackBreakpoint`, which tells them at which breakpoint should they appear stacked on top of each other. The `RadioStackBreakpoint` can be added using the `stack` input (see table below).

### LgRadioGroupComponent

#### Inputs

| Name    | Description                                                                    | Type                  | Default                               | Required |
|---------|--------------------------------------------------------------------------------|-----------------------|---------------------------------------|----------|
| `id`    | HTML ID attribute, auto generated if not provided                              | `string`              | `lg-radio-group-id-${nextUniqueId++}` | No       |
| `name`  | Set the name value for all inputs in the group, auto-generated if not provided | `string`              | `lg-radio-group-${nextUniqueId++}`    | No       |
| `value` | Set the default checked radio button, must match the value of the radio button | `boolean` \| `string` | `null`                                | No       |
| `focus` | Set the focus on the fieldset                                                  | `boolean`             | `null`                                | No       |
| `stack` | Stack the segment buttons at a given breakpoint                                | `'md'` \| `'lg'`      | `false`                               | No       |

### LgRadioButtonComponent

#### Inputs

| Name              | Description                                                                                                                   | Type      | Default                             | Required |
|-------------------|-------------------------------------------------------------------------------------------------------------------------------|-----------|-------------------------------------|----------|
| `id`              | HTML ID attribute, auto generated if not provided                                                                             | `string`  | `lg-radio-button-${++nextUniqueId}` | No       |
| `name`            | HTML name attribute                                                                                                           | `string`  | `null`                              | Yes      |
| `value`           | HTML value attribute                                                                                                          | `string`  | `null`                              | Yes      |
| `ariaDescribedBy` | HTML ID for the corresponding element that describes the radio, if not provided it will use the hint field where appropriate  | `boolean` | `null`                              | No       |

#### Outputs

| Name   | Description                             | Type                  | Default | Required |
|--------|-----------------------------------------|-----------------------|---------|----------|
| `blur` | Event emitted on blur of a radio button | `EventEmitter<Event>` | n/a     | No       |
