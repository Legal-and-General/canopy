# Button

<p class="standfirst">A Button is an interactive component that allows users to initiate actions within an interface.</p>

![Button component example](docs/button/hero.png)

## Usage

Use a Button to clearly indicate the primary action a user should take on a page or within a component. Buttons must be used for actions such as submitting forms, confirming choices, or progressing through a journey. Always ensure the action is clear and unambiguous to the user.

**How to import:**

```js
import { LgButtonComponent } from '@legal-and-general/canopy';
```

**Basic usage:**

```html
<button lg-button type="button" priority="primary">Button</button>
```

## Priorities

![Button component - button styles](docs/button/priorities.png)

Buttons come in two main priorities - Primary and Secondary. Button colours are governed by the parent’s colour mode and always inherit the appropriate theme automatically.

### Primary button

* Use for the main action on a page or within a section.

* There must only be one primary button per page or view.

**Example:**

```html
<button lg-button priority="primary">Primary action</button>
```

### Secondary button

* Use for secondary or less prominent actions.

* Multiple secondary buttons are allowed per page.

**Example:**

```html
<button lg-button priority="secondary">Secondary action</button>
```

## Anatomy

Buttons support several content and icon configurations:

**Label-only**

![Button - button priorities](docs/button/label-only.png)

The default button with text only. Button labels must be concise, action-oriented, and clearly describe the outcome. Avoid vague terms like “Click here” or “Submit.” Use specific verbs that set expectations, such as “Save changes” or “Download report,” and keep labels short - ideally no more than three words.

```html
<button lg-button>Label</button>
```

**Back icon button**

![Button component - back icon button](docs/button/back-icon-button.png)

A button with a back icon followed by a label.

Use the `backIcon` boolean input to display an `arrow-left` icon on the left side of the button. When `backIcon` is set to `true`, it takes precedence and will display the arrow-left icon regardless of any other icons in the markup.

```html
<button lg-button type="button" backIcon="true">
  Back
</button>
```

**Note:** The `backIcon` input specifically displays an arrow-left icon and is intended for navigation purposes. You cannot use a custom icon with this input.

**Right icon button**

![Button component - right icon button](docs/button/right-icon-button.png)

A button with a label followed by an icon to the right, supporting custom icons.

```html
<button lg-button type="button">
  Label
  <lg-icon name="chevron-right" />
</button>
```

**Icon-only button**

![Button component - icon only button](docs/button/icon-only-button.png)

A button with only an icon. You must include visually hidden text for accessibility.

```html
<button lg-button type="button" iconButton="true" >
  Add item
  <lg-icon name="add" />
</button>
```

**Loading state**

![Button component - loading state](docs/button/loading-state.png)

Shows a loading spinner and disables the button.

```html
<button lg-button [loading]="true">Loading…</button>
```

## Dos and Don'ts

### Do

![Button component - dos](docs/button/do.png)

1. **Do** use concise, descriptive labels that clearly communicate the button’s action.
2. **Do** use buttons only for clear, actionable tasks—such as submitting forms, confirming choices, or progressing journeys.
3. **Do** use a single primary button per page or view to indicate the main action.
4. **Do** use full-width buttons on mobile breakpoints for easier tapping.
5. **Do** provide visually hidden text for icon-only buttons to support screen readers.
6. **Do** group related buttons using the Button Group component for consistent alignment and spacing.

### Don't

![Button component - don'ts](docs/button/dont.png)

1. **Don't** place more than one primary button on a page or in a single view.
2. **Don't** use too many buttons in close proximity, which can overwhelm users.
3. **Don't** use labels that are wordy, vague, uninformative, or generic - for example “Click here,” “Submit,” or similar.
4. **Don't** wrap button text labels onto multiple lines.
5. **Don't** centre or right-align buttons unless the layout specifically requires it—left alignment is preferred for consistency.
6. **Don't** use a button for navigation if a link is more appropriate (e.g., for moving between pages).
7. **Don't** use full-width buttons at large (LG) or wider breakpoints—full-width is only appropriate on mobile or small screens.

## Additional development details

### LgButtonComponent inputs

|                | Description                                                                                                                                     | Type                      | Default        | Required |
|----------------|-------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------|----------------|----------|
| `priority`      | The variant of button: `primary`,`secondary`, `link`, `add-on`.                                   | `string`                  | `primary` | No       |
| `fullWidth`    | If the button has to span full width or not. For 'sm' and 'md' sized screens, the button will always be full width and this input has no affect | `boolean`                 | `false`        | No       |
| `disabled`     | Programmatically disable the button via this property                                                                                           | `boolean`                 | `false`        | No       |
| `loading`      | If the button shows a loading spinner and is also disabled                                                                                      | `boolean`                 | `false`        | No       |
| `backIcon`     | Display arrow-left icon on the left side of the button                                                                                          | `boolean`                 | `false`        | No       |
| `iconButton`   | The button displays an icon only                                                                                                                | `boolean`                 | `false`        | No       |

### Styling links as buttons

In almost all cases, use a standard `<button>` element for actions. Only use an `<a>` element styled as a button when a link is required for semantic or technical reasons—such as when the action must navigate to a new page or route, and cannot be achieved with a button.

If you must style a link as a button, use the implementation shown in the example below. For accessibility and best practice, refer to the [Link component documentation](./?path=/docs/components-link-guide--docs).

```html
<a lg-button href="#" priority="primary">Link</a>
```

### Button group

Buttons can be grouped together by using the `LgButtonGroupComponent` – a container to group together two or more buttons and place them along a single line.

Useful for forms where a 'Submit' and 'Cancel' button are placed side by side, for example.

### Button toggle directive

A button toggle is a button used to show or hide content, such as opening and closing a filter panel. Toggle buttons use distinct styles from regular buttons and must use a `<button>` element for accessibility. Using any other element (such as a link or div) will throw an error.

> **Note:** The button toggle directive should only be used as part of a pattern, such as toggling the visibility of a filter panel.

Add the `lgButtonToggle` directive to a `<button>` element alongside the standard `lg-button` directive:

```html
<button lg-button lgButtonToggle type="button" priority="primary">Button</button>
```

In your wrapper component, set the `id` and `ariaControls` properties on the toggle button. Use the `toggleActive` output to update the state of the panel you want to show or hide:

```js
@ContentChild(forwardRef(() => LgButtonToggleDirective)) buttonToggle: LgButtonToggleDirective;

...

ngAfterContentInit(): void {
  this.buttonToggle.id = '<the-unique-id-of-the-toggle>';

  this.buttonToggle.ariaControls = '<the-unique-id-of-the-panel-to-toggle>';

  this.subscription = this.buttonToggle.toggleActive.subscribe(isActive => {
    // set the active state of the panel e.g.:
    // this.myPanelComponent.isActive = isActive;
  });
}
```

For a complete example of a toggle button controlling a panel inside a container, see the [filter container story](./?path=/docs/patterns-filter-container-guide--docs).
